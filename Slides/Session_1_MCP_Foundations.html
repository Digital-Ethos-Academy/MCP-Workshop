What is MCP?

The Model Context Protocol (MCP) is an open standard that enables AI applications to connect with external data sources and tools through a unified interface.

ğŸ”Œ "USB-C for AI"

A standardized way to plug different capabilities into any AI application

Before MCP

  • Custom integrations for each AI app
  • Different APIs, different formats
  • Rebuild tools for each platform
  • No standard security model

With MCP

  • Build once, use everywhere
  • Standard protocol for all AI apps
  • Consistent security model
  • Growing ecosystem of servers

Payments Industry Example

Imagine building a Transaction Search MCP Server once, then using it with:

  • VS Code + GitHub Copilot for developer queries
  • Claude Desktop for analyst investigations
  • Custom internal tools for support teams

One server, multiple AI applications, consistent interface!

Key Components

  • Host: The AI application (VS Code, Claude Desktop)
  • Client: Connects to MCP servers (managed by host)
  • Server: Exposes tools, resources, and prompts
  • Transport: Communication layer (stdio, SSE, HTTP)

How It Works

  1. 1. Host starts and connects to configured servers
  2. 2. Servers announce their capabilities
  3. 3. User asks AI to do something
  4. 4. AI calls appropriate server tools
  5. 5. Results flow back to the user

MCP vs REST APIs

Aspect REST API MCP
Primary User Developers writing code AI applications
Discovery Read documentation Auto-discovered by AI
Descriptions For humans (docs) For AI (docstrings, schemas)
State Stateless Can maintain session state
Communication Request/Response Bidirectional, streaming

Key Insight: MCP is designed for AI-to-service communication, while REST APIs are designed for developer-to-service communication. MCP servers have rich descriptions that help AI understand when and how to use each tool.

MCP Security & Best Practices

MCP servers have significant access to your systems. Evaluate before you install!

Security Evaluation Checklist

Criteria What to Check
Source Code Is it open source? Can you audit it?
Author Reputable organization? Active maintainer?
Permissions What does it access? Files? Network? Credentials?
Activity Recently updated? Active community?
Reviews What do others say about it?

Trust Boundaries

Understand what each server can access:

  • Filesystem servers: Read/write files in specified directories
  • API servers: Make network requests on your behalf
  • Database servers: Execute queries against your data

Red Flags

  • No source code available
  • Requests excessive permissions
  • No documentation or unclear purpose
  • Abandoned or unmaintained project
  • Negative community feedback

PCI-DSS Considerations for Payments

When building MCP servers for payments systems, consider:

  • Data Access: Never expose full card numbers - use tokens
  • Logging: Log tool calls for audit trails
  • Authentication: Ensure proper auth before exposing data
  • Encryption: Use TLS for any network transport
  • Scope: Limit servers to specific, necessary functions

Building MCP Servers with FastMCP

FastMCP is the Python SDK that makes building MCP servers simple using decorators.

Args: transaction_id: The unique transaction identifier (e.g., TX-12345) Returns: Transaction details including amount, merchant, and status """ # Your implementation here return {"id": transaction_id, "amount": 100.00, "status": "completed"} # Run the server if __name__ == "__main__": mcp.run()
